# Project Scope Statement

**Project Name:** Personal Portfolio Website

## Project Purpose

> *Why are we doing this project?*

This project is something I've actually wanted to do for a while but never had a real reason to start! The goal is to build a two-page personal portfolio website that showcases who I am, my interests, and the work I do in my free time, specifically my digital art and programming projects.

Having an *actual* portfolio site is something I think matters for anyone going into a creative or technical field; and this project gave me the push to finally make one rather than just have a GitHub profile sitting there with no real landing page attached to it.

## Description

> *How would you describe this project?*

This project will be a simple two-page personal portfolio website built with semantic HTML and an external CSS file. My main inspiration is the portfolio site of illustrator and developer [Sharlene Yap](https://www.sharyap.com)! I want to include *some* personality with a clean layout, though obviously scaled way down to match what's realistic for this course.

This website will...

- Include an `index.html` homepage with a brief introduction on me and my interests, along with links to my GitHub and relevant profiles in the navigation or footer
- Include an `about.html` page with a more in-depth look at my artwork
- Use a single external `style.css` file that gives it a lively and fun vibe
- Be housed in the *root* directory of a public GitHub repository

## Desired Results

> *What specific outcomes must we achieve to be successful?*

- A valid `index.html` that uses semantic structural tags (`<header>`, `<main>`, `<footer>`, etc.)
- A valid `about.html` that links back to the homepage and goes deeper into my work!
- A `style.css` file that applies consistent styling across both pages
- A `docs/` folder containing `scope.md`, `plan.md`, and `retrospective.md`
- A `images/` folder containing any images I'd like to use for the website
- A `README.md` detailing the purpose of the project along with linking to both the live site and the docs folder
- Everything submitted as a single public GitHub repository before the deadline

## Exclusions

> *What is out of the project's scope and should **NOT** be included?*

- JavaScript or *complex* interactive functionality
  - Basic enhancements like image embeds are permitted as enrichment, but the core requirements come first!
- A full multi-page site beyond the required two pages
- Any backend components, databases, or hosting beyond GitHub Pages
- Features that go beyond what the Project 03 rubric *explicitly* asks for
